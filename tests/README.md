# Каталог tests

Автоматизированные сценарии проверки структурированные по подсистемам и объектам продукта в виде тестов кодом.

Содержимое каталога — скриптовые тесты OneScript (`*.os`), запускаются через `1testrunner -run tests/ИмяТеста.os` (или OneUnit).

Бинарные файлы тестов здесь не хранятся:

* дымовые наборы Vanessa-ADD поставляются в составе пакета `add` (`oscript_modules/add/tests/smoke`) и запускаются по пути из `env.json` (`testsPath: $addRoot/tests/smoke`);
* собственные тестовые обработки 1С хранятся исходниками в `src/tests`, собранные из них `.epf` попадают в `build/out/tests` (артефакт сборки, в git не хранится). Сборка: `vrunner compileepf src/tests build/out/tests` или команда «Собрать unit тесты» расширения 1C: Platform Tools.
